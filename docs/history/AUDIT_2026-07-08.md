## Edge-Smear Fixes Now In `edge_smear_fix`

These are experimental and default off. They do not use black strips, blur, overscan, guard bands, or hidden crop insets.

1. Submitted projection FOV is matched exactly to the ViewLab-cropped FOV returned from `xrLocateViews`.
2. Submitted `subImage.imageRect` is only corrected when it is invalid or out of tracked swapchain bounds.
3. Recommended image rect width/height are aligned to 4-pixel boundaries where possible.
4. When original runtime FOV is known, recommended image scaling uses real per-eye tangent range instead of assuming symmetric left/right and top/bottom FOV.

Verbose diagnostics now compare original runtime FOV, cropped locate FOV, submitted projection FOV before/after, `subImage.imageRect`, swapchain size, recommended size, crop settings, visibility-mask state, visor state, and Direct C release/late-draw provenance.

## Static Bug Scan Results

- Fixed incomplete visor preview/runtime agreement: Direct C now chooses closed bean or open-inner geometry based on the same `Stencil outer edges only` setting that controls the UI preview.
- Fixed Apex Y clamping consistency: global, per-app, preview, and DLL all use `-0.5..0.5`.
- Fixed profile preview context: per-app profile windows receive the current open-inner preview mode and disable `Inner low` when that shape is not active.
- Fixed ReShade Remote corrupted glyph labels by replacing them with ASCII button labels.
- WPF Release build now passes with 0 warnings.
- Native x64 and Win32 Release builds pass with 0 warnings in targeted MSBuild runs.
- `Tests/Verify-ViewLabContracts.ps1` verifies the visor/edge-smear settings contract and passes.

## Known Limits

- Build/tests do not prove headset appearance. The visor and edge-smear changes still need in-headset confirmation.
- The FOV-weighted recommended-size correction can only use original runtime FOV after a locate call has occurred. If a game enumerates and creates swapchains before any locate call, the first swapchain size uses the previous fallback scale until the game re-enumerates.
- ReShade Remote proves payload presence and shared-memory heartbeat behavior, not effect rendering. A game launch test is still required.
- A/B visor source paths remain in source as historical/forensic paths but are not product-facing.

## Headset Test Checklist

1. Install the latest MSI.
2. Launch one known Direct C-working game.
3. Enable `Visor mask`, leave Technique implicit Direct C, set Size around `0.82`, Curve high, `Stencil outer edges only` on.
4. Move `Apex Y` from `0` to `0.25`, then `-0.25`; confirm the outer curve apex moves vertically.
5. Move `Inner low` from `0` to `0.20`; confirm only the lower inner edge gains a curved stencil.
6. Disable `Stencil outer edges only`; confirm preview changes to closed bean and `Inner low` no longer affects the active shape.
7. Test edge smear with `edge_smear_fix=0`, then `1`, at horizontal `0.8`, `0.6`, `0.4`, and vertical around `0.20`/`0.15`.
8. Enable `verbose_logging=1` only for short capture runs; collect `ViewLab.verbose.log`.
9. Compare visible smear with verbose `edge-smear contract` lines for FOV, imageRect, swapchain, and recommended-size mismatch.

## Optimization / Future Work Queue

- Build a small geometry unit test harness for `BeanMaskEditor` vs native visor geometry so preview/runtime shape drift can be caught without a headset.
- Add a one-click log bundle button that copies config, latest main log, latest verbose log, and app profile registry export into a zip for edge-smear diagnosis.
- Add a visible status line for ReShade install state: payload bundled, ProgramData installed, OpenXR registry enabled, heartbeat live.
- Consider a proper "debug capture mode" that temporarily enables verbose edge-smear diagnostics for one launch and then turns itself off.
- If edge smear persists, investigate compositor/runtime-specific sampling behavior with controlled `imageRect == swapchain` and visual-mask-only comparison captures.
- Keep one-sided edge masks disabled until there is a real native implementation; do not let legacy keys silently enable broad masks.
